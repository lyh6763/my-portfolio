# SNS 앱 기획안

> **프로젝트 안내**
> - 데이터베이스는 Supabase MCP를 사용해서 구현 예정
> - 현재 프로토타입 프로젝트이기 때문에 이메일 인증이나 보안 기능들을 사용하지 않음

---

# 1부: 페이지 설계

## 1. 로그인 페이지
- 로고
- 아이디 입력란
- 비밀번호 입력란
- 로그인 버튼 (메인 페이지로)
- 비밀번호 찾기
- 회원가입 버튼

### 1-1. 회원가입 페이지
- 유저 이름 입력란
- 아이디 입력란
- 비밀번호 입력란
- 비밀번호 확인 입력란
- 회원가입 완료 버튼 (로그인 페이지로 복귀)

### 1-2. 비밀번호 찾기 페이지
- 페이지 타이틀
- 뒤로가기 버튼
- 아이디 입력란
- 새 비밀번호 입력란
- 새 비밀번호 확인 입력란
- 비밀번호 변경 완료 버튼

---

## 2. 메인 페이지 (피드/타임라인)

### 2-1. 상단바
- 로고
- 알림 아이콘 (상태변화 - 알림 표시; 붉은 점) -> 알림 페이지로 이동
- 메시지 아이콘 -> 메시지 페이지로 이동

### 2-2. 피드 (타임라인)
- 게시물(피드)의 요소
	- 프로필 영역
		- 프로필 이미지
		- 이름(게시자 이름)
		- 위치/장소
		- ...메뉴 (프로필 보기, 신고, 차단)
	- 이미지 영역
		- 이미지 슬라이드
		- 슬라이드 하단 순서 UI (도트형)
	- 리액션 영역
		- 좋아요, 댓글, 공유, 저장
		- 좋아요 수
		- 캡션 (게시물 내용 - 한줄, 더보기)
		- 댓글 수 + 모두보기 -> 댓글 페이지로 이동
		- 게시물 업로드 시간

### 2-3. 하단바 (fixed)
- 홈 (메인 페이지-피드로 복귀)
- 검색 (검색 페이지로 이동)
- 게시물 업로드 (업로드 페이지로 이동)
- 프로필 (프로필 페이지로 이동)

---

## 3. 알림 페이지
- 헤더
	- 페이지 타이틀
	- 뒤로가기 버튼
- 알림 목록
	- 내 게시물 좋아요 알림 -> 클릭 시 해당 게시물로 이동
	- 나를 팔로우 알림 -> 클릭 시 해당 유저 프로필로 이동
	- 내 댓글에 답글 알림 -> 클릭 시 해당 댓글로 이동
	- 알림 읽음/읽지 않음 표시
	- 알림 시간 표시
- 빈 상태: 알림이 없을 때 안내 메시지

---

## 4. 메시지 페이지

### 4-1. 대화 목록 페이지
- 헤더
	- 페이지 타이틀
	- 뒤로가기 (메인 페이지로 이동)
	- 새 메시지 / 설정 / 검색
- 대화 리스트
	- 상대 프로필 이미지
		- 온라인 활동 중 표시
	- 이름
	- 마지막 메시지 미리보기
	- 타임스탬프
	- 읽지 않음 표시 (읽지 않은 메시지 수)
	- 고정 / 뮤트 표시
- 빈 상태: 대화가 없을 때 안내 메시지

### 4-2. 채팅 상세 페이지
- 헤더
	- 뒤로가기 버튼
	- 상대방 프로필 이미지 + 이름
	- 온라인 상태 표시
	- 더보기 메뉴 (알림 설정, 차단, 신고)
- 채팅 영역
	- 날짜 구분선
	- 상대방 메시지 (좌측 정렬)
	- 내 메시지 (우측 정렬)
	- 메시지 시간 표시
	- 읽음 표시
	- 이미지/미디어 메시지
- 입력 영역
	- 텍스트 입력란
	- 이미지 첨부 버튼
	- 전송 버튼

---

## 5. 검색 페이지
- 상단 검색란
	- 검색어 입력
	- 검색 버튼 / 취소 버튼
- 검색 전: 추천 피드 (3열 그리드)
- 검색 결과
	- 탭: 유저 / 게시물 / 해시태그
	- 유저 검색 결과: 프로필 이미지, 이름, 팔로워 수
	- 게시물 검색 결과: 3열 그리드
	- 해시태그 검색 결과: 태그명, 게시물 수
- 빈 상태: 검색 결과 없을 때 안내 메시지

---

## 6. 게시물 업로드 페이지

### 6-1. 사진 선택 페이지
- 헤더
	- 뒤로가기 버튼
	- 페이지 타이틀
	- 다음 버튼
- 선택된 사진 미리보기
- 갤러리 그리드 (4열)
- 다중 선택 기능 (최대 9장)
- 선택 순서 표시

### 6-2. 게시물 작성 페이지
- 헤더
	- 뒤로가기 버튼
	- 페이지 타이틀
	- 업로드 버튼
- 선택된 이미지 미리보기
- 글 작성란 (500자 이내)
- 위치 추가
- 해시태그 입력
- 사람 태그

---

## 7. 게시물 상세 페이지
- 헤더
	- 뒤로가기 버튼
	- 더보기 메뉴 (수정, 삭제 - 내 게시물일 경우)
- 게시물 내용 (피드와 동일한 구조)
	- 프로필 영역
	- 이미지 영역
	- 리액션 영역
	- 전체 캡션 표시
- 댓글 영역
	- 댓글 목록
	- 댓글 입력란

### 7-1. 댓글 페이지 (바텀시트)
- 드래그 바 (아래로 내려 닫기)
- 댓글 목록
	- 프로필 이미지
	- 이름
	- 댓글 내용
	- 작성 시간
	- 좋아요 버튼 + 수
	- 답글 달기 버튼
	- 답글 목록 (접힘/펼침)
- 댓글 입력란
	- 내 프로필 이미지
	- 텍스트 입력
	- 게시 버튼

### 7-2. 게시물 수정 페이지
- 헤더
	- 취소 버튼
	- 페이지 타이틀
	- 완료 버튼
- 이미지 미리보기 (수정 불가)
- 글 수정란
- 위치 수정
- 해시태그 수정

## 8. 프로필 페이지

### 8-1. 프로필 메인
- 유저 이름
- 프로필 이미지
- 자기소개
- 게시물 수
- 팔로워 수 -> 팔로워 목록 페이지로 이동
- 팔로잉 수 -> 팔로잉 목록 페이지로 이동
- (내 프로필) 프로필 편집 / 프로필 공유 / 설정 버튼
- (타인 프로필) 팔로우 버튼 / 메시지 버튼
- 게시물 영역 (3열 그리드)
- 저장됨 탭 (내 프로필만) -> 저장한 게시물 그리드

### 8-2. 프로필 편집 페이지
- 헤더
	- 취소 버튼
	- 페이지 타이틀
	- 완료 버튼
- 프로필 이미지
	- 이미지 변경 버튼
- 이름 입력란
- 사용자 이름 입력란
- 자기소개 입력란
- 링크 입력란

### 8-3. 팔로워 목록 페이지
- 헤더
	- 뒤로가기 버튼
	- 페이지 타이틀 (팔로워)
- 검색란
- 유저 목록
	- 프로필 이미지
	- 이름
	- 사용자 이름
	- 팔로우/팔로잉 버튼
- 빈 상태: 팔로워가 없을 때 안내 메시지

### 8-4. 팔로잉 목록 페이지
- 헤더
	- 뒤로가기 버튼
	- 페이지 타이틀 (팔로잉)
- 검색란
- 유저 목록
	- 프로필 이미지
	- 이름
	- 사용자 이름
	- 팔로잉 버튼 (언팔로우)
- 빈 상태: 팔로잉이 없을 때 안내 메시지

### 8-5. 저장된 게시물 페이지
- 헤더
	- 뒤로가기 버튼
	- 페이지 타이틀
- 저장된 게시물 그리드 (3열)
- 빈 상태: 저장된 게시물이 없을 때 안내 메시지

# 2부: 데이터베이스 설계

> 기술 스택: Supabase (PostgreSQL)

## 테이블 목록 (총 15개)

| # | 테이블명 | 설명 |
|---|----------|------|
| 1 | users | 사용자 정보 |
| 2 | posts | 게시물 |
| 3 | post_images | 게시물 이미지 (최대 9장) |
| 4 | hashtags | 해시태그 목록 |
| 5 | post_hashtags | 게시물-해시태그 연결 |
| 6 | comments | 댓글/답글 |
| 7 | likes | 좋아요 (게시물/댓글) |
| 8 | follows | 팔로우 관계 |
| 9 | saved_posts | 저장된 게시물 |
| 10 | blocks | 차단 관계 |
| 11 | conversations | 대화방 |
| 12 | conversation_participants | 대화 참여자 |
| 13 | messages | 메시지 |
| 14 | notifications | 알림 |
| 15 | notification_settings | 알림 설정 |

## 주요 테이블 구조

### users (사용자)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| username | VARCHAR(50) | 아이디 (로그인용) |
| password | VARCHAR(255) | 비밀번호 |
| display_name | VARCHAR(100) | 표시 이름 |
| profile_image | TEXT | 프로필 이미지 URL |
| bio | VARCHAR(300) | 자기소개 |
| link | TEXT | 프로필 링크 |
| is_private | BOOLEAN | 비공개 계정 여부 |
| created_at | TIMESTAMP | 가입일 |

### posts (게시물)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → users.id |
| caption | VARCHAR(500) | 게시물 내용 |
| location | VARCHAR(100) | 위치/장소 |
| created_at | TIMESTAMP | 작성일 |

### post_images (게시물 이미지)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| post_id | UUID | FK → posts.id |
| image_url | TEXT | 이미지 URL |
| order_index | INTEGER | 이미지 순서 |

### comments (댓글)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| post_id | UUID | FK → posts.id |
| user_id | UUID | FK → users.id |
| parent_id | UUID | FK → comments.id (답글) |
| content | VARCHAR(500) | 댓글 내용 |
| created_at | TIMESTAMP | 작성일 |

### follows (팔로우)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| follower_id | UUID | 팔로우 하는 사용자 |
| following_id | UUID | 팔로우 받는 사용자 |
| created_at | TIMESTAMP | 팔로우 일시 |

### messages (메시지)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| conversation_id | UUID | FK → conversations.id |
| sender_id | UUID | FK → users.id |
| content | TEXT | 메시지 내용 |
| image_url | TEXT | 이미지 URL |
| created_at | TIMESTAMP | 전송 시간 |

### notifications (알림)
| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | UUID | PK |
| user_id | UUID | 알림 받는 사용자 |
| actor_id | UUID | 알림 발생시킨 사용자 |
| type | VARCHAR(20) | like, comment, follow, reply |
| post_id | UUID | 관련 게시물 |
| is_read | BOOLEAN | 읽음 여부 |
| created_at | TIMESTAMP | 알림 시간 |

## ERD 관계 요약

```
users ──1:N──► posts ──1:N──► post_images
  │              │
  │              └──1:N──► comments (self-reference for replies)
  │              │
  │              └──1:N──► likes
  │
  ├──N:M──► follows (follower ↔ following)
  │
  ├──N:M──► blocks (blocker ↔ blocked)
  │
  ├──N:M──► saved_posts
  │
  └──N:M──► conversations ──1:N──► messages
```
